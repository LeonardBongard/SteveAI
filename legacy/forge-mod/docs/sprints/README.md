# Sprint Archive

This directory is the canonical archive for sprint planning, review, and retrospective records.

## Rules
1. Every new sprint gets its own file.
2. Use the template in `SPRINT_TEMPLATE.md`.
3. Use `SPRINT_CHECKIN_TEMPLATE.md` for lightweight review/retrospective updates during the sprint.
4. Keep one sprint marked `Active` at a time.
5. When a sprint ends, update its review/retrospective sections instead of rewriting history elsewhere.
6. All sprint files must tie back to:
- `docs/SCRUM_OPERATING_MODEL.md`
- `docs/ROADMAP_NORTH_STAR.md`
- `docs/EXECUTION_BOARD.md`
- `docs/PRODUCT_BACKLOG.md`

## Index
1. `SPRINT_0001_RUNTIME_REPLANNING_SPINE.md`
- Status: Active
- Sprint Goal: Steve can detect no-progress and perform structured runtime replanning without stalling.

## Naming Convention
Use:
- `SPRINT_0001_<SHORT_NAME>.md`
- `SPRINT_0002_<SHORT_NAME>.md`

Use uppercase file names with underscores so the archive stays easy to scan.
